About multiProcessing



multProcessing creates multiple ACE editors (embeddable code editor written in javascript) with multiple Processing livecode outputs into your HTML.

It is a tools designed to help document Processing codes on web pages, which also allows them to be interactive.


Created by Prof. Guilherme Ranoya in 2019 under a General Public License (GPLv3) while writing his online book on creative technologies (in portuguese).

Source code.

Dependencies



multiProcessing is dependent on ACE Editor and Processing.js

They are inserted in the HTML from the first 7 lines of code in the multiProcessing source, both loaded from cdnjs.cloudflare.com.

multiProcessing Examples



Simplest implementation

A pre element with processingeditor class and a targetId property pointing to a iframe element id.



<script src="https://www.ranoya.com/Assets/JSLibs/multiProcessing/multiProcessing.js"></script>

<pre class='processingeditor editorstyle' targetId="SketchOutput1">
void setup() {
size(150,150);
background(#78008A);
frameRate(20);
}

void draw() {
fill(#FFFFFF);
rect(20,20,100,100);
}
</pre>

<iframe id="SketchOutput1" class="outputstyle" frameborder=0></iframe>
void setup() {
    size(150,150);
    background(#78008A);
    frameRate(20);
}

void draw() {
    fill(#FFFFFF);
    rect(20,20,100,100);
}

Implementation with predata and postdata

The same as the simplest form, but using predata_[targetId] for code that must be put before what is inside the editor (pre element), and postdata_[targetId] for what must be after (don't forget to close open brackets!). Usefull for locking or hidding parts of the code.



<script src="https://www.ranoya.com/Assets/JSLibs/multiProcessing/multiProcessing.js"></script>

 <script>
 predata_SketchOutput2 = `
 void setup() {
 size(1000,1000);
 background(#78008A);
 stroke(255);
 strokeWeight(10);
 }

 void draw() {
 `;

 postdata_SketchOutput2 = `
 rect(10,10,10,10);
 }
 `;
 </script>

<pre class='processingeditor editorstyle' targetId="SketchOutput2">
line(20,20,400,300);
rect(50,50,40,40);
</pre>

<iframe id="SketchOutput2" class="outputstyle" frameborder=0></iframe>
line(20,20,400,300);
rect(50,50,40,40);

Changing ACE Editor setup and styling blocks

Use data-linguagem to set the right code highlight and linter, data-acetheme to set color and editor style, and data-config in the pre element to set options of the ACE Editor.

Access Aces Kitschen Sink to try all the editor's properties, and this Github Page for how to write setup options. 

<script src="https://www.ranoya.com/Assets/JSLibs/multiProcessing/multiProcessing.js"></script>

<pre class='processingeditor editorstyle bigger' data-linguagem="javascript"
data-acetheme="cobalt" data-config='{"highlightActiveLine":true,
"selectionStyle":"line", "foldStyle":"markbeginend",
"readOnly":true}'>
void setup() {
size(150,150);
background(#78008A);
frameRate(20);
}

void draw() {
fill(#FFFFFF);
rect(20,20,100,100);
}
</pre>

You can set global preferences for all editors in the page with the acegeneralconfig object:


<script>
acegeneralconfig = {
maxLines: Infinity,
enableBasicAutocompletion: true,
enableSnippets: true,
enableLiveAutocompletion: false,
showPrintMargin: false,
highlightSelectedWord: true,
selectionStyle: "text",
highlightActiveLine: false,
cursorStyle: "wide",
wrapBehavioursEnabled: true,
wrap: true
};
</script>


Only elements with processingeditor class will be transformed into an ACE Editor. You can use this class for styling, or can add other classes to the class property of the element for better styling.
